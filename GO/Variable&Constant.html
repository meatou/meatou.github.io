# 变量

所谓的变量就是一个拥有指定名称和类型的数据存储位置。

示例：标准写法

// 语法：
var 变量名 数据类型 = 变量值
// 例子：
var x string = "hello world"

示例：先定义变量，再赋值

// 语法：
var 变量名 数据类型
变量名 = 变量值
// 例子：
var x string 
x = "hello world"

示例：Go语言推断变量的类型

// 语法：
var 变量名 = 变量值
// 例子：
var x = "hello world"

示例：变量定义的快捷方式

// 语法：
变量名 := 变量值
// 例子：
x := "hello world"

注意

上面这种使用:=方式定义变量的方式只能用在函数内部。

提示

变量之所以称为变量，就是因为它们的值在程序运行过程中可以发生变化，但是它们的变量类型是无法改变的。因为Go语言是静态语言，并不支持程序运行过程中变量类型发生变化。比如如果你强行将一个字符串值赋值给定义为int的变量，那么会发生编译错误。即使是强制类型转换也是不可以的。强制类型转换只支持同类的变量类型。比如数值类型之间强制转换。

变量命名规则

Go的变量名称必须以字母或下划线（ _ ）开头，后面可以跟字母，数字，或者下划线( _ )。除此之外，Go语言并不关心你如何定义变量。我们通用的做法是定义一个用户友好的变量。假设你需要定义一个狗狗的年龄，那么使用dog_age作为变量名称要好于用x来定义变量。

变量作用域

所谓作用域就是可以有效访问变量的区域。比如很简单的，你不可能在一个函数func_a里面访问另一个函数func_b里面定义的局部变量x。所以变量的作用域目前分为两类，一个是全局变量，另一个是局部变量。

# 常量

所谓常量就是在程序运行过程中保持值不变的变量定义。常量的定义和变量类似，只是用const关键字替换了var关键字，另外常量在定义的时候必须有初始值。

# 定义

示例：标准写法

// 语法：
const 常量名 数据类型 = 常量值
// 例子：
const x string = "hello world"

示例：先定义常量，再赋值

// 语法：
const 常量名 数据类型
常量名 = 常量值
// 例子：
const x string 
x = "hello world"

示例：Go语言推断常量的类型

// 语法：
const 常量名 = 常量值
// 例子：
const x = "hello world"

注意

变量定义的类型推断方式:=不能够用来定义常量。因为常量的值是在编译的时候就已经确定的，但是变量的值则是运行的时候才使用的。这样常量定义就无法使用变量类型推断的方式了。

常量的值在运行过程中是无法改变的，强制改变常量的值是无效的。

# 多变量或多常量定义

示例：多变量定义

// 语法：
var (
    变量名1 数据类型 = 变量值1
    变量名2 数据类型 = 变量值2
    变量名3 数据类型 = 变量值3
    ...
)

// 例子：
var (
    a int = 10
    b float64 = 32.45
    c bool = true
)

示例：多常量定义

// 语法：
const (
    常量名1 数据类型 = 常量值1
    常量名2 数据类型 = 常量值2
    常量名3 数据类型 = 常量值3
    ...
)

// 例子：
const (
    Pi float64 = 3.14
    True bool = true
)
